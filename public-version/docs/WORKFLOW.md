# Octopus Hunter Workflow

## Development Process

1. Identify the task.
2. Review the existing implementation.
3. Propose a plan.
4. Wait for approval.
5. Implement the change.
6. Self-review the code.
7. Present the changes.
8. Human review.
9. Merge.
10. Update CHANGELOG_AI.md.

## Rules

- One feature at a time.
- One logical commit per feature.
- Preserve backward compatibility.
- Do not modify unrelated files.
- No speculative refactoring.
- No breaking changes.
- Every change must be reviewable.

## Definition of Done

A task is complete only when:

- Code compiles.
- Existing functionality is preserved.
- Required migrations are provided.
- Testing steps are documented.
- Human approval is received.
- CHANGELOG_AI.md has been updated.
